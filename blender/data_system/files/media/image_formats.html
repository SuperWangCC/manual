通道深度¶

图像文件格式支持不同数量的每像素位数。这影响色彩的质量和文件大小。

常用的深度:

8 bit (256 levels)

最常见的屏幕上的图形和视频

10, 12, 16 bit (1024, 4096, 65536 levels)
Used for some formats focusing on photography and digital films (such as DPX and JPEG 2000).
16 位半浮点

由于完整的 32 位浮点数通常是超过足够的精度，那么半浮法可以节省同时提供高动态范围的磁盘空间。

32 位浮点数

高质量颜色深度。

Blender内置的图像系统支持其中任一:

  • 每个通道 (4 × 8 位) 的 8 位。

  • 每个通道 (4 x 32 位)-32 位浮点 * 使用尽可能多的 memory.* x 4

高于 8 位 / 通道的图像会转换成漂浮在加载到搅拌机。

注解

浮点数经常用于: 词: ‘HDRI’，

当图像有浮颜色，在搅拌器默认要使用的所有影像功能。这包括视频序列编辑器、 纹理映射、 背景图像和排序。

格式细节¶

DPX和Cineon——Annita翻译¶

Cineon是柯达的胶片数码化标准，10位/通道，对数运算原理。DPX已经从Cineon衍生出名为ANSI / SMPTE的行业标准。DPX支持16位色/通道，线性以及对数。DPX标准目前被广泛应用于电影的软硬件行业。

DPX以及Cineon只储存和转换数值在0.0到1.0之间的“可见“色彩范围（如渲染或混合的结果）。没有通道被复写。代码已经顺利的从CinePaint中复制出来。据CinePaint的主要开发者罗宾所言，DPX代码默认为对数的色彩范围。我们还没找出禁用这的方法，但它似乎读/写正常，没有明显的损失。Blender DPX文件（整个《大象之梦》电影）已经顺利通过英国宽泰（公司）的IQ/高清/2K/剪辑/分级标准的考验，被成功引进，所以我们眼下认为它适合为专业人士所使用。

OpenEXR——Annita翻译¶

ILM’s OpenEXR has become a software industry standard for HDR image files, especially because of its flexible and expandable structure.

OpenEXR文件可以存储多层和多个通道。意味着 OpenEXR 图像可以在合成编辑器中完好无损的保持渲染层和通道。

渲染输出:¶

可选的OpenEXR渲染输出方式有:

Color Depth

把图像存储在自定义的每通道16位的浮点格式中。这使实际的“位深”降到10位，以及带5个字节的能量值和1个字节的符号值。

Float (Half), Float (Full)

Codec
PIZ (无损)
Lossless wavelet compression. Compresses images with grain well.
ZIP (无损)
Standard lossless compression using Zlib.
RLE (无损)
Run-length encoded, lossless, works well when scanlines have same values.
Pxr24 (有损)
Lossy algorithm from Pixar, converting 32 bits floats to 24 bits floats.
Z Buffer

保存深度信息。Blender现在也是写在浮动值里，通过“Blender单位长度”显示与摄像机的精确距离。

预览

在渲染动画（或者通过命令行渲染单帧）时，Blender也会将同一张图片保存为JPEG格式，加快预览及下载的速度。

Radiance HDR¶

Radiance 最初是由格雷格 • 沃德写的照明模拟工具套件。Radiance 最先 (和长一段时间唯一)使用 HDR 图像格式，此格式由许多其他软件包支持。

Radiance (.hdr) 文件存储颜色仍然 8 位中的每个组件，但与附加 (共享) 8 位的指数值，使它每像素 32 位。只有 RGB 值可以存储在这些文件中。只有RGB可以存储在这些文件中。
